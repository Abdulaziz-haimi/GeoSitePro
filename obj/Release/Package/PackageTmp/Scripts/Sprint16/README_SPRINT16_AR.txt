GeoSitePro - Sprint 16
======================

اسم المرحلة:
Security Hardening & Deployment Readiness

الهدف:
تقوية طبقة الأمان والتجهيز العملي قبل نشر النظام على بيئة إنتاج.

الصفحات الجديدة:
1) SecurityCenter.aspx
   - لوحة أحداث الأمان.
   - إحصائيات آخر 7 أيام.
   - تسجيل حدث أمني يدوي.
   - عرض أحداث فشل الدخول والأحداث الحرجة.

2) PasswordPolicy.aspx
   - سياسة كلمة المرور.
   - طول كلمة المرور.
   - متطلبات الحروف والأرقام والرموز.
   - انتهاء كلمة المرور.
   - عدد محاولات الدخول الفاشلة.
   - مدة قفل الحساب.
   - انتهاء الجلسة.

3) DeploymentChecklist.aspx
   - قائمة تحقق قبل نشر الإنتاج.
   - بنود أمان وقاعدة بيانات ونسخ احتياطي واختبار وتشغيل.
   - حساب نسبة جاهزية الإنتاج.

63_Create_Permissions_Seed_Sprint16.sql
64_Create_StoredProcedures_Sprint16.sql

ملاحظة مهمة:
هذه المرحلة تضيف الهيكل التشغيلي الأمني. تطبيق سياسة كلمة المرور داخل منطق Login.aspx.cs يجب اختباره لاحقًا على Visual Studio حتى لا نكسر تسجيل الدخول الحالي أثناء التطوير.
